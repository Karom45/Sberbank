Структура папки Geography:
ПАПКИ:
Папка Big - в ней происходит обработка крупных сущностей (свыше 100 000 сущностей).
	Подпапка Information идентична корневой папке Information в ней обрабатываются поблочно(50 000 сущностей) крупные файлы
	Подпапка pool служит для загрузки блоков и дальнейшей конкатенации.Струтура идентична Information
	bookquery.py хранит SPARQL-запросы
	concatenating.ipynb  скрипт для объединения блоков , а также для создания файлов типа aggregated_with_popularity
	entity_finder.ipynb схож со скриптом в корне, однако в цикле работает не со всем файлом , а с блоками.
Папка csv - хранит csv и excel файлы.
	Analytics - необработанный файл с финальной аналитикой
	Entities содержит типы сущостей , а также их количество , ранжированное по значимости типа
	Properties содержит все предикаты wikidata
	qid_popularity содержит ID и популярность сущностей из wikidata.Выгружен с Igraph
	Raw_analytics является шаблоном для аналитики.Содерржит все типы сущностей.
Final analytics хранит итоговую аналитику в файле Analytics.
For_aggregation:
	just_aggregation - скрипт для агрегации информации на основе собранных предикатов(создание файлов aggregated и aggregated_with popularity
Information содержит всю информацию по предикатам:
	Папки 1 и 2 хранят информацию соответсвенно о типах сущностей со значимость 1 и 2
		data хранит csv файлы в в виде ID - название(либо ID-ID для крупных)
			data with popularity хранит csv файлы в виде ID-название-популярность, где популярность сумма входящих и исходящих ссылкок на ссущность(ID-ID-популярность для крупных)
			rus_labels(только в 2) содержит русские лейблы для сущностей имеющих оные 
		analyzing содержит csv-файлы вида ID-название-название предиката - 1(наличие) - количество данных по предикату - направление(входящее или исходящее)
			aggregated содержит csv-файлы вида название предиката-общее количество самих предикатов- общее количество данных внутри предиката - процент встречаемости среди всех
			aggregated_with_popularity содержит csv-файлы структурно идентичные файлам в aggregated , однако олько по самым популярным сущностям (5 %)
json - папка для обработки json
	в корне лежит json , а в папки result - преобразованный csv
Processed_predicates папка для хранения обработанных предикатов.Содержит файлы типа aggregated и aggregated_with_popularity с произведенной экспертной оценкой по значимости предикатов
СКРИПТЫ:
all_predicates - скрипт формирующий файл Properties.csv , содержайщий все предикаты.
cheking_popularity - скрипт, сравнивающий популярность дял найденных сущностей с популярностями для сущностей из файла qid_popularity
csv_to_excel преобразовывет csv в excel и добавляет поле signifance
entity_analyzer - скрипт , создающий файл Analytics
entity_finder - скрипт, формирующий все файлы в Information(выборка предикатов,популярностьей, предикатов и дальнейшая группировка)
JSON_to_csv - преобразует json в csv
predicate_selector - обрабатывает файлы с предикатами и отбрасывает технические или неудовлетворяющие условиям задачи
rus_labels -скрипт для вытягивания русских названий для сущностей
